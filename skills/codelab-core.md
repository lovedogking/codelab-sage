# Codelab 核心编程价值观

## 1. 可读性优先

- 命名要表达意图，优先让代码自我解释，而不是依赖注释去说明“这段代码在做什么”。
- 函数应只做一件事，名称直接体现这件事。
- 避免缩写、魔法数字、无意义的变量名（如 `a`、`x`、`data`）。

## 2. 不要过早抽象

- 三个相似用例出现之前，不要引入抽象。
- 宁可先写一点重复代码，也不要为了“未来可能的变化”制造复杂的继承链或配置层。
- 抽象的标准是：它让调用者更简单，而不是让维护者更困惑。

## 3. 明确处理错误

- 异步操作必须处理异常，不要吞掉错误。
- 返回结果应让调用者一眼知道成功还是失败。
- 错误信息要对人友好，对机器可解析。

## 4. 最小改动原则

- 只改与任务直接相关的代码。
- 不要为了“顺便优化”扩大 PR 范围。
- 重构和修复 bug 应该分成独立的提交或 PR。

## 5. 尊重项目既有约定

- 新代码要像旧代码一样：遵循现有的命名、目录结构、注释密度和错误处理方式。
- 不确定时，先查看同目录下已有文件的风格。

## 6. 工具是为任务服务，不是炫技

- CLI 的每一行输出都应有明确目的。
- 不要引入不必要的依赖；引入新依赖时必须说明不可替代的理由。
